你/妳好，我是邱昱儒，也可以叫我 Louis， 畢業於元智大學電機工程系乙組 (通訊系)， 現職是一位軟體工程師，於今年9月年資屆滿一年， 工作內容主要是使用 JavaScript 搭配 React 撰寫前端及 Java 搭配 Spring Boot 開發後端。

在校期間由於本系選課上給的彈性較大提供多元修課路線，在修習完系上必修後， 個人修課內容大多偏向人工智慧，但由於大三時臨近畢業開始有就業壓力， 而礙於師資等緣故以及系上人工智慧相關課程逐年減少，加上參考網路上建議， 普遍對於機器學習、深度學習及資料科學領域的技能樹需求較不明確或是過於廣泛。 當時的自己軟體開發技能較為零散，以及對機器學習與深度學習的了解不夠深入， 因此在大三、大四期間上網透過 Yotube, Udemy 以及各種開放線上資源自學， 累積軟體開發能力，轉換目標變為朝向成為前、後端工程師。

大四上修習行動裝置開發課時，因緣際會收到了教授的研究計畫邀約，專案內容主題是監測餐飲油煙對於 TVOC 指標的影響， 我負責內容如下：

  • 建立收集資料程式，從感測器接收並存入資料庫
  • 建立監測網頁提供畫面查看監測指標值
  • 將資料依照教授提供策略撰寫程式並產生報表供分析

雖然因為畢業未能完整參與研究，但我從中獲得了對資料工程與基礎資料分析的初步認識。

在成為軟體工程師後，也持續思反思己的職涯方向，上網閱讀了許多人的心得、分享等，了解了很多軟體相關職涯， 並藉由近期 LLM 的崛起，了解了許多人工智慧的應用，再次發覺自己對資料科學領域仍有興趣， 期望可以藉此機會參加 Wehelp 深度學習 Bootcamp 更加完善自己深度學習基礎知識， 並於結訓後可以轉換職涯跑道，回到自己當初有興趣資料科學領域。

曾經做過哪些軟體工程技術相關的學習？若有作品請分享給我們。

空氣品質監控工具

承第一題回答，曾經參與系上研究計畫，主要涉及了：

  • 資料工程：搭建簡易資料管道，感測器透過 MQTT 協定傳輸並收集至資料庫。使用工具為 Node.js。
  • 前、後端網頁開發：搭建監控網站，查詢資料庫的監測資料，顯示柱狀圖。使用工具為 JS、React 及 Express 等。
  • 資料分析：依照教授需求將資料進行處理，並產生報告供分析。使用工具為 Python、Pandas 等。

如果參與這個訓練，會怎麼安排學習時間？

由於公司是接案公司，本身風氣較為自由，且相較其他接案公司，我們 loading 也相較輕鬆，若錄取預計是仍會在職學習，應會是線上參與。 且本身下班後及假日一直都有持續進修學習的習慣，自認為不會因為工作關係而投入較少心力及時間。 若順利申請上了，也會依照 bootcamp 規劃進度進行學習，因為本身對深度學習算是有粗淺概念，若完成每週分派進度後仍有餘裕也願意將時間投入進一步自行進修。

軟體技術日新月異，如何確定選擇投入的領域是正確有回報的？

自己對於職涯的方向仍然是不太明確，但我認為在職涯初期多接觸各方知識並無壞處，即使接觸多方的選擇相較專精一個領域可能會走得比較慢，但只要遵循自己的步伐走，也不見得有有什麼正確、錯誤的投入，在確認自己的興趣或職涯後再精進也不乏是一個選擇。 畢竟軟體開發其實有許多概念是互通的，也許可以將曾在某個領域的投入應用於現在的領域中。

請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

當自己遇到負面情緒時，會轉而去做自己感興趣的事，轉換念頭後讓自己放鬆後，再重新面對自己產生這個負面情緒的源頭。

在成為軟體工程師後有保持著持續學習的習慣，由於本身在學時資料結構、演算法並沒有講得很深入， 因此在進修這部分知識時曾經懷疑過自己，陷入一陣子的低潮，開始懷疑自己是否擁有軟體領域需要的持續學習的能力。

後來因為對最近很紅的語言 Rust 感興趣，因於放下正在進修的資料結構演算法，改而學習 Rust 重拾起學習的熱情， 也是這時候才覺得走得比較慢也沒關係，讓自己持續學習也許才是重點。

如何看待自身工作和整個社會群體的連結關係？

自己算是對社會關注度沒有那麼高，主要接觸的資訊大多還是藉由朋友中的閒聊得知。 且工作上開發的專案大多是對企業內部系統，工作上對社會群體的連結關係可能較弱。

但身為軟體工程師，也一直對於貢獻開源專案抱有興趣，這對我不僅是一種技術挑戰， 也許是另一種將技術回饋給社會的方式，但了解後發現自己的程式撰寫能力尚不足， 因此仍在持續學習如 Clean Code 等開發原則，希望有朝一日可以貢獻開源專案。
